Note: the embed page above is rendered using the same snippet you'd paste into your CMS. The only difference is that data-pi-url="" is left empty here, which makes the JS default to the current origin (the Pi). When pasting elsewhere, set data-pi-url to your Pi's Tailscale Funnel or Cloudflare Tunnel URL.

How to use

  1. In streamer.toml, set [server] public_streams = true and restart the service.
  2. Confirm the Pi is reachable on a public URL (Tailscale Funnel works out of the box for personal use).
  3. View source on this page and copy everything between the ===== SNIPPET BEGIN and ===== SNIPPET END markers.
  4. Paste into a WordPress Custom HTML block (or any plain HTML container).
  5. Edit data-pi-url="" on the mount <div> to point at your Pi's public URL, e.g. data-pi-url="https://hedgebuggy.tailfoo.ts.net".

What it does

  • Polls /api/public/status every 20 s (configurable via data-poll-interval-ms) to detect sleep state.
  • During AWAKE: renders both camera tiles with live MJPEG streams.
  • During ENTERING_SLEEP: keeps streams live and shows a yellow banner counting down to sleep.
  • During ASLEEP (or whenever the Pi is offline): replaces tile contents with a dimmed card showing the next wake time. The hosted page itself stays reachable.
  • Overnight (Pi fully powered off, status endpoint unreachable): uses a cached copy of the sleep/wake schedule (saved in localStorage on every successful status poll) to show "asleep until ~HH:MM" instead of a generic offline message. Visitors whose browser has never reached the Pi see the offline card instead.
