Single-camera embed

One widget per camera page — snapshot and fullscreen controls, sleep-schedule awareness, and overnight “asleep until …” cards when the Pi powers down. Use a separate embed on each WordPress page (cam0 page, cam1 page, etc.).

Live preview

Camera number is inferred from the URL when you open /embed/cam0 or /embed/cam1. On this generic page, set data-camera on the mount <div> below.

Legacy dual-tile embed: the older two-camera snippet at /embed still works but is no longer recommended for new pages.

WordPress (recommended)

Copy the wordpress/ folder from this repo into wp-content/plugins/hedgework-cam-embed/ and activate the plugin. Then add a shortcode to each camera page: 

[hedgework_cam camera="0" pi_url="https://your-pi.tailfoo.ts.net"]
[hedgework_cam camera="1" pi_url="https://your-pi.tailfoo.ts.net"]

The plugin loads embed-cam.css and embed-cam.js from its own directory — no inline script in your page, so WordPress entity-encoding is not an issue.

Custom HTML (upload CSS + JS)

Upload embed-cam.css and embed-cam.js (from src/streamer/webui/ or /static/ on the Pi) to your WordPress media library or theme. Paste into a Custom HTML block on each camera page — edit paths and data-camera per page: 

<link rel="stylesheet" href="/wp-content/uploads/embed-cam.css" />
<div
  class="hedgework-cam-embed-single"
  data-pi-url="https://your-pi.tailfoo.ts.net"
  data-camera="0"
  data-poll-interval-ms="20000"
></div>
<script src="/wp-content/uploads/embed-cam.js"></script>

Setup

  1. Set [server] public_streams = true in streamer.toml and restart the service.
  2. Confirm the Pi is reachable on a public URL (Tailscale Funnel, Cloudflare Tunnel, etc.).
  3. One embed per camera page — set data-camera="0" or "1" (or use the shortcode camera attribute).
  4. Strip trailing slashes from data-pi-url / shortcode pi_url (the JS tolerates them, but cleaner URLs help).

Controls

  • Snapshot — saves the current MJPEG frame as a JPEG download (enabled once the first frame arrives).
  • Fullscreen — expands the video stage to fill the screen.
  • Watch live — appears after a stream ends (server time limit, sleep, or disconnect). Status polls do not reopen a forgotten tab; the visitor must click to resume.

Sleep behaviour

  • Polls /api/public/status every 20 s (configurable via data-poll-interval-ms).
  • ENTERING_SLEEP — yellow banner with countdown; stream stays live.
  • ASLEEP or unreachable Pi overnight — asleep card with next wake time (uses localStorage schedule cache when the Pi is powered off).
